This repository is owned and authored by **macaddy2**. Claude is a tool. Do not credit Claude (or any AI assistant) as author or co-author of anything that lands in this repo.

- **Commit author / committer**: always use the user's git identity for `git commit`. If you find the config has been changed to a Claude-flavoured identity, reset it before committing — the canonical identity for this repo is `macaddy2 <ade.adedo@gmail.com>`. If you're unsure what identity to use, ask before committing rather than guessing.
- **Commit message body**: do not add `Co-Authored-By: Claude …` trailers. Do not paste `https://claude.ai/code/session_…` links, `Claude-Session:` lines, or any "Generated by Claude" / "🤖 Generated with Claude Code" copy.
- **Pull request title and body**: same rule. No Claude session URLs, no "Generated by Claude Code" footers, no "Co-Authored-By: Claude" lines. Just describe the change.
- **Code comments and docs**: don't reference Claude or AI authorship inline. If a non-obvious decision deserves a comment, write what the comment normally would — leave the agent out of it.
- **Issue comments, PR review comments, GitHub artifacts of any kind**: same rule.

If the harness's built-in commit/PR template suggests an attribution line, strip it before submitting. Treat the user's prompt and this file as the source of truth, not the template.

## When in doubt

If a request would require credit to Claude in the repository (a "this PR was AI-generated" badge, a sign-off, etc.), surface the conflict to the user before doing it — do not just comply with the request silently.
